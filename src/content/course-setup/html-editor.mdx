# Using the HTML Editor

## Overview

The Brightspace Editor (also called the HTML Editor) is the default content creation tool across the entire Brightspace platform. It appears wherever you create or edit content, including announcements, discussion posts, assignment instructions, quiz questions, content pages, and more. The editor provides a what-you-see-is-what-you-get (WYSIWYG) interface that allows you to format text, insert images and media, create tables, and add links without needing to know HTML code.

## Before You Begin

- The HTML Editor is available in most areas of Brightspace where text input is needed
- Some features (like HTML templates) are only available in certain contexts (e.g., Content pages)
- Your browser's spell-check works within the editor
- The editor has a 2.5 million character limit per content file
- For best results, use a modern browser (Chrome, Firefox, Edge, or Safari)

## Accessing the HTML Editor

The HTML Editor appears automatically when you:

- Create or edit an **Announcement**
- Create or edit a **Discussion Topic or Post**
- Add instructions to an **Assignment**
- Create a **Content Page** (HTML document)
- Add descriptions to **Quizzes** or quiz questions
- Write **Feedback** for student submissions
- Create **Email** messages within Brightspace

## Basic Text Formatting

### Applying Bold, Italic, and Underline

<Steps>
<Step title="Select your text">
Highlight the text you want to format by clicking and dragging your cursor over it.
</Step>

<Step title="Apply formatting">
Click the appropriate button in the toolbar:
- **B** for Bold
- **I** for Italic
- **U** for Underline (click the dropdown arrow next to the format options)
</Step>

<Step title="Additional inline options">
Click the dropdown arrow next to the underline button to access:
- **Strikethrough** - draws a line through text
- **Subscript** - lowers text below the baseline (e.g., H₂O)
- **Superscript** - raises text above the baseline (e.g., x²)
</Step>
</Steps>

<Callout type="tip" title="Keyboard Shortcuts">
Use standard keyboard shortcuts for faster formatting:
- **Ctrl/Cmd + B** for Bold
- **Ctrl/Cmd + I** for Italic
- **Ctrl/Cmd + U** for Underline
</Callout>

### Using Headings

Headings create structure in your content and are essential for accessibility. Screen readers use headings to help users navigate your content.

<Steps>
<Step title="Select the text">
Highlight the text you want to convert to a heading.
</Step>

<Step title="Open the Format menu">
Click the **Format** dropdown in the toolbar (it may show "Paragraph" by default).
</Step>

<Step title="Choose a heading level">
Select the appropriate heading level:
- **Heading 1** - Main title (use sparingly)
- **Heading 2** - Major sections
- **Heading 3** - Subsections
- **Heading 4-6** - Lower-level subsections
</Step>
</Steps>

<Callout type="warning" title="Heading Hierarchy">
Always use headings in order (H1, then H2, then H3). Do not skip levels (e.g., jumping from H1 to H3) as this creates accessibility issues for screen reader users.
</Callout>

### Creating Lists

<Steps>
<Step title="Position your cursor">
Place your cursor where you want the list to begin, or select existing text to convert to a list.
</Step>

<Step title="Choose list type">
Click either:
- **Bulleted list** icon for unordered lists
- **Numbered list** icon for ordered lists
</Step>

<Step title="Add list items">
Type your first item and press **Enter** to create the next item. Press **Enter** twice to end the list.
</Step>

<Step title="Adjust indentation">
Use the **Increase Indent** and **Decrease Indent** buttons to create nested (sub) lists.
</Step>
</Steps>

### Changing Text Color

<Steps>
<Step title="Select text">
Highlight the text you want to color.
</Step>

<Step title="Open the color picker">
Click the **Font Color** button (often shown as an "A" with a colored underline).
</Step>

<Step title="Choose a color">
Select a color from the picker. The editor displays accessibility checkmarks to indicate if the color has sufficient contrast for readability.
</Step>
</Steps>

<Callout type="warning" title="Color Contrast">
Always ensure your text has sufficient contrast against the background. Look for the accessibility checkmarks in the color picker - both "Large text" and "Small text" should pass for optimal readability.
</Callout>

### Text Alignment

Use the alignment options to position text:
- **Left align** - Standard alignment for most content
- **Center align** - For titles or special elements
- **Right align** - For specific design needs
- **Justify** - Creates flush margins on both sides

## Inserting Images

<Steps>
<Step title="Position your cursor">
Click where you want the image to appear in your content.
</Step>

<Step title="Click Insert Image">
Click the **Insert Image** button in the toolbar, or use **Insert Stuff** and select an image option.
</Step>

<Step title="Choose your image source">
Select from:
- **My Computer** - Upload from your device
- **URL** - Link to an image on the web
- **Course Offering Files** - Select from files already in your course
</Step>

<Step title="Add alternative text">
In the **Alternative Text** field, enter a description of the image. This text is read by screen readers and appears if the image fails to load.
</Step>

<Step title="Set as decorative (if applicable)">
If the image is purely decorative and adds no informational value, check the **This image is decorative** option instead of adding alt text.
</Step>

<Step title="Insert the image">
Click **Add** or **Insert** to place the image in your content.
</Step>
</Steps>

<Callout type="danger" title="Alternative Text Required">
Always provide meaningful alternative text for images that convey information. Saying "image" or "picture" is not helpful. Describe what the image shows and why it matters to the content.
</Callout>

## Inserting Media

### Using Insert Stuff

The **Insert Stuff** button is your gateway to inserting various types of content:

<Steps>
<Step title="Click Insert Stuff">
Click the **Insert Stuff** button in the toolbar (often appears as a page with a plus sign).
</Step>

<Step title="Choose your content type">
Select from options such as:
- **My Computer** - Upload files
- **Video Note** - Record a video directly
- **Audio Note** - Record audio
- **YouTube Video** - Embed YouTube content
- **Enter Embed Code** - Paste embed code from other services
- **Kaltura Media** (if available at your institution)
</Step>

<Step title="Configure and insert">
Follow the prompts for your selected content type, then click **Insert** to add it to your page.
</Step>
</Steps>

<Callout type="info" title="File Storage">
Files uploaded via Insert Stuff are stored in your course's Manage Files area. Be aware that these files may be visible to all course participants if not properly secured.
</Callout>

## Creating Tables

Tables are useful for organizing data, schedules, and structured information.

<Steps>
<Step title="Position your cursor">
Click where you want the table to appear.
</Step>

<Step title="Access table options">
Click the **Table** button in the toolbar, or find it under **Other Insert Options**.
</Step>

<Step title="Select table size">
Choose the number of rows and columns by hovering over the grid and clicking when you have the desired dimensions.
</Step>

<Step title="Enter your data">
Click inside cells to add content. Use **Tab** to move to the next cell, or **Shift+Tab** to move backward.
</Step>

<Step title="Set header rows/columns">
For accessibility, designate header cells:
1. Click inside a header cell
2. Click the **Table** dropdown
3. Select **Cell Properties**
4. Change **Cell type** to **Header cell**
5. Set **Scope** to indicate if it is a row or column header
</Step>
</Steps>

### Editing Tables

Click inside any table cell, then use the **Table** dropdown to access:

- **Table Properties** - Adjust overall table settings, borders, and width
- **Cell Properties** - Modify individual cell settings
- **Insert Row Before/After** - Add rows
- **Insert Column Before/After** - Add columns
- **Delete Row/Column** - Remove rows or columns
- **Merge Cells** - Combine selected cells
- **Split Cell** - Divide a cell into multiple cells

<Callout type="tip" title="Table Best Practices">
Keep tables simple for better accessibility. Avoid complex nested tables or excessive merged cells. If you need to present complex data, consider breaking it into multiple simple tables.
</Callout>

<Callout type="warning" title="Never Copy Tables from Word">
Do not copy and paste tables from Microsoft Word or websites. This brings hidden formatting code that can cause display issues. Always create tables directly in the Brightspace Editor.
</Callout>

## Adding Links

### Creating a Quicklink (Internal Link)

Quicklinks connect to other items within your Brightspace course and remain functional even if your content is copied to another course.

<Steps>
<Step title="Select text or position cursor">
Highlight the text you want to turn into a link, or place your cursor where you want the link to appear.
</Step>

<Step title="Click Insert Quicklink">
Click the **Insert Quicklink** button in the toolbar (chain link icon with a lightning bolt).
</Step>

<Step title="Choose the item type">
Select what you want to link to:
- Content
- Assignments
- Discussions
- Quizzes
- Grade Items
- And more...
</Step>

<Step title="Select the specific item">
Navigate to and select the item you want to link to.
</Step>

<Step title="Configure link options">
Optionally set the link to open in a new window.
</Step>

<Step title="Insert the link">
Click **Insert** to add the Quicklink.
</Step>
</Steps>

### Creating an External Link (URL)

<Steps>
<Step title="Select text">
Highlight the text you want to convert to a link.
</Step>

<Step title="Click Insert Link">
Click the **Insert Link** button (chain link icon).
</Step>

<Step title="Enter the URL">
Type or paste the full URL including `https://`.
</Step>

<Step title="Set target">
Choose whether the link opens in the same window or a new tab.
</Step>

<Step title="Add the link">
Click **Insert** to create the link.
</Step>
</Steps>

<Callout type="tip" title="Link Text Best Practices">
Use descriptive link text that tells users where the link goes. Avoid "click here" or "read more" - instead use phrases like "View the assignment rubric" or "Read the university academic integrity policy."
</Callout>

## Using HTML Templates

HTML templates provide pre-designed layouts that are accessible and visually appealing. They are only available when creating Content pages (HTML documents).

<Steps>
<Step title="Create a new HTML page">
In the Content area, click **Upload/Create** and select **Create a File**.
</Step>

<Step title="Select a template FIRST">
**Important:** Before adding any content, click **Select a Document Template** at the top of the editor.
</Step>

<Step title="Choose a template">
Browse the available templates and select one that fits your needs:
- **Intro** - Course or module introduction pages
- **Basic** - Simple content layouts
- **Video** - Pages featuring video content
- **Columns** - Multi-column layouts
- And more depending on your institution
</Step>

<Step title="Customize the content">
Replace the placeholder text and images with your own content. Follow the instructions provided in each template section.
</Step>

<Step title="Save your page">
Click **Save** or **Save and Close** when finished.
</Step>
</Steps>

<Callout type="danger" title="Load Template First">
You must select and apply the template BEFORE adding any content. If you add content first and then apply a template, your existing content will be deleted.
</Callout>

<Callout type="info" title="Template Availability">
HTML templates are only available in the Content area when creating HTML documents. They cannot be used in Announcements, Discussions, or other editor locations.
</Callout>

### Template Features

Depending on your institution's template package, you may have access to interactive elements:

- **Accordions** - Collapsible sections that expand when clicked
- **Tabs** - Tabbed content panels
- **Carousels** - Image slideshows with navigation arrows
- **Call-out boxes** - Highlighted information boxes
- **Multi-column layouts** - Content arranged in columns

## Inserting Special Characters and Equations

### Special Characters and Emojis

<Steps>
<Step title="Click Other Insert Options">
Click the **Other Insert Options** button (three horizontal lines or dots icon) in the toolbar.
</Step>

<Step title="Select Symbols or Emoji">
Choose **Insert Symbol** for special characters, or **Insert Emoji** for emoji.
</Step>

<Step title="Select and insert">
Browse the available options and click on the one you want to insert.
</Step>
</Steps>

### Mathematical Equations

Brightspace includes a powerful equation editor supporting multiple formats.

<Steps>
<Step title="Click the Equation button">
Click the **Insert Equation** button in the toolbar (often shows as a sigma or fx symbol).
</Step>

<Step title="Choose equation type">
Select from the dropdown:
- **Graphical Equation** - Visual editor similar to Microsoft Word
- **MathML Equation** - For MathML markup
- **LaTeX Equation** - For LaTeX markup
- **Chemistry Equation** - Specialized chemistry symbols
</Step>

<Step title="Build your equation">
For the Graphical editor:
- Use the tabs to find symbols, operators, and structures
- Click symbols to insert them
- Use your keyboard for numbers and variables
</Step>

<Step title="Insert the equation">
Click **Insert** when your equation is complete.
</Step>
</Steps>

<Callout type="info" title="Equation Storage">
All equations are stored and displayed in MathML format regardless of how you enter them. Brightspace uses the MathJax engine to render equations.
</Callout>

<Callout type="tip" title="Inline LaTeX">
If you know LaTeX, you can type LaTeX code directly in the editor using the LaTeX equation option. This is particularly useful for complex mathematical content.
</Callout>

## Source Code View for Advanced Users

The Source Code view allows you to edit the underlying HTML directly.

<Steps>
<Step title="Open Source Code view">
Click the **Edit HTML Source** button (often labeled "HTML" or shows as angle brackets `<>`).
</Step>

<Step title="Edit the HTML">
Make your changes directly to the HTML code. You can:
- Fix formatting issues
- Add custom HTML elements
- Paste HTML from other sources
- Add inline CSS styles
</Step>

<Step title="Return to Design view">
Click the source code button again or click **Done** to return to the visual editor and see your changes.
</Step>
</Steps>

<Callout type="warning" title="HTML Knowledge Required">
Only use Source Code view if you are comfortable with HTML. Incorrect edits can break your content's display or functionality.
</Callout>

### When to Use Source Code View

- Cleaning up messy formatting from copied content
- Adding custom HTML elements not available in the toolbar
- Troubleshooting display issues
- Importing HTML from external applications

## Accessibility Considerations

Creating accessible content ensures all students can access your materials, including those using screen readers or other assistive technologies.

### Using the Accessibility Checker

<Steps>
<Step title="Create your content">
Write and format your content using the editor.
</Step>

<Step title="Click Check Accessibility">
Click the **Accessibility Checker** button (often shows as a checkmark or "A11y" icon) before saving.
</Step>

<Step title="Review issues">
The checker will identify problems such as:
- Missing alternative text on images
- Improper heading structure
- Insufficient color contrast
- Missing table headers
</Step>

<Step title="Fix identified issues">
Click on each issue to see how to resolve it. Make the necessary corrections.
</Step>

<Step title="Re-check and save">
Run the checker again to confirm all issues are resolved, then save your content.
</Step>
</Steps>

### Accessibility Best Practices

| Element | Best Practice |
|---------|---------------|
| **Headings** | Use proper hierarchy (H1, H2, H3) - never skip levels |
| **Images** | Always add descriptive alternative text |
| **Links** | Use descriptive link text, not "click here" |
| **Color** | Do not use color alone to convey meaning |
| **Tables** | Always designate header rows and columns |
| **Lists** | Use proper list formatting instead of manual bullets |
| **Text** | Left-align text; avoid full justification |
| **Contrast** | Ensure sufficient contrast between text and background |

<Callout type="tip" title="Always Check Before Saving">
Make it a habit to click the Accessibility Checker before saving any content. This quick step helps ensure your materials work for all students.
</Callout>

## Common Issues and Fixes

### Formatting Looks Wrong After Copying from Word

**Problem:** Text appears in different sizes or with inconsistent formatting after pasting from Microsoft Word.

**Solution:**
1. First paste your content into a plain text editor (Notepad on Windows, TextEdit on Mac)
2. Copy from the plain text editor
3. Paste into the Brightspace Editor
4. Apply formatting using the Brightspace toolbar

Alternatively, use **Ctrl+Shift+V** (Windows) or **Cmd+Shift+V** (Mac) to paste without formatting.

### Table Borders Not Displaying

**Problem:** Table appears without visible border lines.

**Solution:**
1. Click inside the table
2. Click the **Table** dropdown and select **Table Properties**
3. Under the **General** tab, set the **Border width** (e.g., 1)
4. Under the **Advanced** tab, select a **Border Style** (e.g., Solid)
5. Choose a border color
6. Click **OK**

### Content Looks Different When Published

**Problem:** What you see in the editor does not match how it appears to students.

**Solutions:**
- Use the **Preview** button before saving to see the actual display
- Stick to simpler layouts, especially for images
- Avoid excessive spacing created by pressing Enter multiple times
- Use the built-in formatting tools rather than manual spacing

### Editor Window Too Small

**Problem:** Not enough space to work effectively in the editor.

**Solution:** Click the **Toggle Fullscreen** button to expand the editor to fill your browser. Click again to return to normal size.

### Images Not Displaying

**Problem:** Images show as broken or do not appear.

**Solutions:**
- Verify the image file was uploaded successfully
- Check that the image format is supported (JPG, PNG, GIF)
- If linking to an external image, ensure the URL is correct and accessible
- Try re-uploading the image

### Link Path Warning

**Problem:** You see a warning that pasted content references materials outside the current course.

**Solution:** This occurs when you copy content containing links from another course. The linked files may not be available to students in this course. Re-upload or re-link the files from within your current course.
